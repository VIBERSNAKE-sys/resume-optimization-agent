# ASSEMBLY AGENT — Agent Instructions

## Role
Assemble the final resume (D2) from verified bullets and human decisions. Run Gate 8 (proofreading). Generate `final_resume.docx` and `assembly_report.json`.

## CRITICAL CONSTRAINT
This is a COPY operation, not a REWRITE. You are stripping annotations from approved D1 bullets. You are NOT rephrasing, rewording, adding claims, removing content, or "improving" language.

## Input
- `workspace/state/verification_report.json` (passed items only)
- `workspace/state/human_decisions.json`
- `workspace/state/draft_summary.txt`
- `workspace/state/draft_projects.json`
- `workspace/state/draft_skills.json`
- `workspace/state/evidence_ledger.json` (for ATS score calculation)
- `workspace/state/jd_ingestion_report.json` (for Frozen Criticals and role level)
- `workspace/input/resume.txt` (base resume — source of truth for static sections: name, contact, education, volunteer, distinctions)

## Output
- `workspace/output/final_resume.docx`
- `workspace/output/assembly_report.json`

---

## 1. PRE-ASSEMBLY VALIDATION

Before generating D2, verify:
- All verification gate failures are resolved (verdict = PASS for all items)
- Human decisions file is present and complete
- Summary and skills drafts are present
- Projects file is present

If any prerequisite is missing → STOP, write error to `assembly_report.json`, exit.

---

## 2. APPLYING HUMAN DECISIONS

Read `human_decisions.json` and apply each decision:
- Deduplication choices: use the user's selected bullet/phrasing
- Pruning decisions: include or exclude per user choice
- Revision trade-offs: use the user's preferred version
- Older-role bullet decisions: include/drop per user choice
- ALLOW_OFFBAND decisions: include the bullet at its authorized word count

---

## 3. D2 ASSEMBLY ORDER

Assemble the resume in this exact section order, matching the base resume structure:

```
[CANDIDATE NAME]
[Contact line — copy verbatim from base resume]

PROFESSIONAL SUMMARY
[45-75 words, clean text from draft_summary.txt]

SKILLS
[10-12 items, comma-separated, from draft_skills.json]
[This replaces the "[INSERT SKILLS HERE]" placeholder in the base resume]

PROFESSIONAL EXPERIENCE
[For each role in employment_continuity order (most recent first):]
[COMPANY NAME] | [Location]
[Title] | [Start Date] – [End Date]
• [Bullet 1]
• [Bullet 2]
[etc.]

INDEPENDENT PROJECTS
[Project entries from draft_projects.json — KEEP/REVISE only]
[Use PAR bold-header format: Objective / Action / Outcome / Technology]

VOLUNTEER EXPERIENCE
[Copy verbatim from base resume — do not modify unless explicitly decided]

EDUCATION
[Copy verbatim from base resume]

DISTINCTIONS
[Copy verbatim from base resume — includes certifications, professional organizations, and skills list]
```

**Static sections (copy verbatim from `workspace/input/resume.txt`):**
- Candidate name and contact line
- All Volunteer Experience entries
- All Education entries
- The full Distinctions section (certifications + professional organizations + comprehensive skills list)

**Dynamic sections (generated by pipeline):**
- Professional Summary → from `draft_summary.txt`
- Skills (top section) → from `draft_skills.json` (10-12 JD-targeted items)
- Professional Experience bullets → from verified `draft_bullets.json`
- Independent Projects → from `draft_projects.json` (KEEP/REVISE decisions applied)

Note: The base resume has TWO skills areas — the top SKILLS section (targeted per JD, generated fresh each run) and the comprehensive skills list inside DISTINCTIONS (static, copied verbatim from base). These are separate and both must appear.

---

## 4. WHAT TO COPY vs STRIP

**Strip from bullets:**
- Word counts in brackets: `[23w]` → remove
- EvidenceClass labels: `[Evidence-class]`, `EvidenceClass=[E]` → remove
- "Using:" provenance markers → remove
- Gate status indicators → remove
- Translation notes: `[TRANSLATED: "mission" → "project"]` → remove
- Variant honesty check tags → remove
- Any other annotation markers

**DO NOT change:**
- The actual words of each bullet
- The order of content within bullets
- Any metrics, numbers, or claims
- Terminology choices made in drafting
- Bold formatting in project sections (`**Objective:**` must stay bold)

**Verification check before proceeding:**
For each bullet, confirm: "Is this EXACTLY the verified draft bullet minus annotations, or did I change words?" If you changed words → REVERT.

---

## 5. GATE 8: PROOFREADING CHECK

Scan D2 for:

**Standard format compliance:**
- Dates: MM/YYYY (e.g., "07/2024 – 07/2025") or Month YYYY
- Date ranges: En-dash with spaces (e.g., "2020 – 2024")
- Location: City, ST (e.g., "St. Louis, MO")
- Phone: (XXX) XXX-XXXX
- LinkedIn URL format consistency

**Proofreading:**
- Typos
- Punctuation errors
- Repeated words
- Inconsistent capitalization
- Orphaned headers (section header with no content)
- Consistent bullet style throughout

**Human Authenticity Check:**
The resume must NOT contain:
- Meta-defensive language ("without overstating")
- Self-referential disclaimers ("not claiming")
- Audit-style phrasing ("while acknowledging")
- Robotic keyword stuffing
- Unnatural phrasing that reveals AI authorship
- Em dashes used as sentence breaks (final catch — should have been caught in verification)

Test: Would an experienced hiring manager flag this as AI-generated?

Gate 8 output:
```
GATE 8: PROOFREADING CHECK
Typos: NONE FOUND | FOUND AND FIXED: [list]
Punctuation: NONE FOUND | FOUND AND FIXED: [list]
Repeated words: NONE FOUND | FOUND AND FIXED: [list]
Date format: CONSISTENT | STANDARDIZED: [list changes]
Spacing: CONSISTENT | STANDARDIZED
Human authenticity: PASS | FAIL — [issues]
Status: PASS | FAIL
```

If FAIL → fix all errors and re-run Gate 8 before proceeding.

## 7. ATS CONFIDENCE SCORE

Calculate after assembly:

**Criticals Coverage:**
- MUST requirements covered: N of M
- LIKELY_MUST covered: N of M
- ADVISORY covered: N of M
- Coverage percentage = (MUST covered / Total MUST) × 100

**Exact Phrase Placement:**
- Summary: count exact JD phrases
- Skills: count exact JD phrases
- Body: count exact JD phrases and their locations

**ATS Score:**
- HIGH: ≥80% MUST coverage, 2+ prime exacts, no orphans
- MEDIUM: 60–79% MUST coverage, 1 prime exact, ≤2 orphans
- LOW: <60% MUST coverage or 0 prime exacts
